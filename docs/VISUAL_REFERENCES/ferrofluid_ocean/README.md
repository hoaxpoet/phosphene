Architecture summary as of round 65:
- **Aurora** (chromatic): `vocals_pitch` → hue, `drums_dev_smoothed` → intensity, `arousal` → drift.
- **Spike heights** (geometry): `bass_dev` → height.
- **Swell amplitude** (atmospheric): `arousal` only.
- **Spike base lattice, wavelengths, speeds, fresnel weight**: constant.

Three layers, three timescales, three primitives. No double-coupling.

> **Stylization caveat for any session reading this folder.** Every photograph in this folder was sourced because it teaches a *specific* trait — material behavior, scale, lattice character, swell motion, palette anchor, atmosphere, or lighting mechanic. None of them is a faithful rendering of "Ferrofluid Ocean as it should appear." Read each image only for the trait its annotation calls out, and *actively disregard* every other property of the image — particularly: studio gel lighting colors; the literal cause of any natural phenomenon depicted (auroras, storms, sun-on-water); foreground objects (lotus veins, foam crests, beach rocks, lightning bolts, mountain silhouettes, lake bottoms, celestial point sources, distant shorelines); any palette that conflicts with §10.3.5's "distant fog cools to dark purple" intent. The preset's hue comes from (a) mood-valence-tinted base sky (D-022 via `scene.lightColor.rgb`) and (b) an audio-reactive aurora curtain in the procedural sky that the substrate mirror-reflects (D-126 paradigm; D-127 routing — a single continuous curtain whose hue is driven by `vocals_pitch_hz` (confidence-gated, mood-valence fallback), whose intensity rides a 150 ms-smoothed `drums_energy_dev` envelope, and whose azimuth drifts at `accumulated_audio_time × arousal`. The curtain lives in the sky function, not as direct lights cast onto the surface). Direct-lit albedo is fixed by §4.6: `float3(0.02, 0.03, 0.05)`. Per-image disregard guidance follows the convention promoted to rule level in **D-065 §(c)** (`SHADER_CRAFT.md §2.1` step 2).

## Mandatory traits (per SHADER_CRAFT.md §12.1)

For this preset specifically, the following implementations are mandatory:

- [ ] **Detail cascade:**
  - **macro** = baked smooth-Voronoi-style spike lattice. `FerrofluidParticles.bakeHeightField` produces a 4096² r16Float height texture by computing per-texel Quilez polynomial smooth-min over 3×3 spatial-hash neighbor cells of 2500 particles laid out on a 50×50 grid with `voronoi_cell_offset` per-cell hash jitter. The SDF samples this texture via bilinear filter. `spike_height ← 1.0 + 0.35 × bass_dev` (round 65) so the lattice has constant base shape with bass-driven height modulation up to 35 % taller on strong transients. Composed on top of a Gerstner-wave macro displacement field (4 superposed waves with Tessendorf horizontal sway per round 62; amplitude from arousal only per round 65). Scope extends to an ocean-portion horizon (see `01_*` for scale, `02b_*` for swell geometry).
  - **meso** = per-cell variation from the `voronoi_cell_offset` particle-position hash; smooth-min blend gives organic non-uniformity by construction. Domain-warp + drums-flow approach retired; smooth-Voronoi approach produces the `02_*` lattice-defect character without an explicit warp seed.
  - **micro** = **deferred / not currently implemented.** The `fbm8(p * 15.0)` normal perturbation and hash-lattice micro-droplets originally specced here were implemented in Session 4 Phase A and reverted in Session 4.5 Phase 0 (Failed Approach #62 — destroyed the substrate's mirror identity, no load-bearing musical role). Future increment may revisit; do not re-implement without explicit re-authorization that articulates the musical role per Authoring Discipline.
  - **specular** = `mat_ferrofluid` (`albedo = float3(0.02, 0.03, 0.05)`, `roughness = 0.08`, `metallic = 1.0`) with Leitl's four-layer `fluid_shading` (round 64 weights: ambient 0.3, fresnel 0.20, specular 0.0, iridescence 0.005). The fresnel layer (round 64 re-enable) provides chrome rim definition on cone silhouettes. Ambient layer reflects the audio-reactive procedural sky via `rm_ferrofluidSky`. Thin-film F0 from `rm_thinfilm_rgb` available for matID==3 (single-light fallback path), not currently routed in matID==2 production path.
- [ ] **Hero geometric function:** baked smooth-Voronoi-style height field over particle positions. Bake kernel: `ferrofluid_height_bake` in `Renderer/Shaders/FerrofluidParticles.metal`. Uses Quilez polynomial smooth-min (`poly_smin`) with `smoothMinW = 0.005` and apex rounding via `almostIdentity` with constants (0.04, 0.016). Spike-base radius 0.17 wu (the round-50 lotus-pattern reduction was reverted in round 55), height multiplier 0.63 in `fo_ferrofluid_field_sampled` → peak aspect ratio 3.65:1 (reference-faithful needle character). The SDF in `sceneSDF` samples the baked height texture, divides by 4 (round 56 Lipschitz correction) to make the height-field SDF Lipschitz-1 conservative — true-distance underestimate that the ray-march can safely step against. References: https://iquilezles.org/articles/smoothvoronoi/, https://iquilezles.org/articles/smin/, https://iquilezles.org/articles/functions/. Tessendorf-style Gerstner sway implemented inline in `fo_gerstner_swell` (4 superposed waves, horizontal steepness 0.3 — round 62).
- [ ] **Material count and recipes:** `mat_ferrofluid` (§4.6, primary spike material) + **thin-film interference layer** via `rm_thinfilm_rgb` in `Shaders/RayMarch.metal` (renderer-local port of `Utilities/PBR/Thin.metal`'s `thinfilm_rgb`; promoted to per-preset mandatory under the V.9 redirect; provides the iridescent shift in aurora-band reflections that reads as "magnetic fluid" rather than chrome). `mat_ferrofluid` from `Shaders/Utilities/Materials/` (V.3). ## Anti-references (failure modes specific to this preset)

What a Claude Code session is most likely to produce by accident; what this preset must NOT look like:

- **Generic chrome metaballs / smooth reflective blobs** with no spike character. The single most likely failure mode if the session shortcut to writing a Beer's-law or simple-displacement surface instead of implementing the §4.6 hex-pack spike field via smooth Voronoi. See `05_anti_chrome_blob_AIGEN.jpg`.
- **Perfectly regular hexagonal lattice.** A risk if the session reverts the smooth-Voronoi cell-hash offsets (which provide the natural per-cell position variation) or reads `01_*`'s apparent regularity literally. The Quilez `voronoi_smooth` utility's `voronoi_cell_offset` hash gives organic non-uniformity by construction; do not "regularize" the cell positions for aesthetic predictability.
- **Radial / hub-and-spoke spike arrangement.** A specific risk if the session over-reads the lotus-leaf vein structure in `03b_*`. Spike centers are hex-pack via smooth Voronoi, not radial. The lotus is in the folder pending re-evaluation of micro-droplet behavior in a future increment (Phase 0 reverted the droplet feature).
- **Foam-capped water as the between-spike base.** A specific risk if the session over-reads the upper portion of `06_*`, the foam crests at the top of `02b_*`, or pattern-matches generic "ocean" imagery. The §4.6 between-spike surface is *mirror-metal*, not water with surf; ferrofluid does not foam.
- **Active weather effects in the fog.** A specific risk if the session over-reads `07_*` and reproduces lightning, dramatic flicker, or rolling cloud movement. The §10.3.5 fog is a static atmospheric tint — tone, not phenomenon.
- **Static surface / missing Gerstner swell.** A specific risk if the session implements the spike-lattice height modulation but forgets the underlying macro Gerstner displacement. The surface must move as a coherent body of liquid, not have spikes pulsing on a flat plane. `02b_*` is the reference for what's missing; absence of swell motion at any time other than total silence is a documented failure.
- **Aurora reads as a club rig** rather than a continuous physical phenomenon. Moving colored bands that strobe on beat onsets in saturated party colors, or band motion that is jittery and beat-quantized rather than continuous and slowly orbital. The §5.8 rig (driving the aurora bands per D-126) produces continuous palette-driven hue rotation and slow orbital motion (orbit speed ← arousal, hue ← `vocals_pitch_hz` normalized inline with `other_energy_dev` fallback, intensity ← `drums_energy_dev` envelope) — never beat-strobed direction changes or onset-edge intensity pulses. See `04_*` for the *quality* of aurora-on-ferrofluid (one frozen instant of mirror-reflected sky); the failure mode is band *behavior* (jittery on-beat strobes, oversaturated party palette).
- **Point-source light reflections / vertical pillar reflections on the surface.** A specific risk if the session reads the bright celestial point in `08_*` as a directive about preset lighting. The aurora bands in the procedural sky are continuous diffuse gradients sampled at the reflection vector (D-126), not sharp point reflections forming vertical pillars (like a moon or lighthouse reflecting in a calm lake). Pillar reflections are exactly the wrong reading and were the failure mode the D-126 paradigm pivot eliminated. See Failed Approach #61.
- **Implementing the §5.8 rig as direct Cook-Torrance point lights with inverse-square attenuation.** Failed Approach #61. At any reasonable physical orbit distance (altitude 6 m / radius 4 m, surface-to-light ~7 m), inverse-square attenuation gives ~0.02× per beam → invisible beams against the IBL the same near-mirror surface reflects. D-126 retired this paradigm in favor of mirror-reflects-procedural-sky: the rig outputs drive aurora bands IN THE SKY, not direct illumination ON THE SURFACE.
- ~~**Calm state still showing spikes.**~~ **RETIRED in round 50 (2026-05-16) under the constant-field premise.** The new silence destination is a calm ferrofluid ocean — full spike lattice present + low-amplitude Gerstner swell + base sky reflection only (no aurora overlay). The `10_silence_calm_body.jpg` anchor now applies to the SWELL portion only (the body-of-liquid motion under the spikes), not to the spike presence question. Implementations should NOT cause the lattice to fade out at low audio.
- **Soft / diffuse highlights.** Anything above `roughness = 0.15` reads as plastic. The §4.6 `roughness = 0.08` is non-negotiable for the "razor-sharp" character. See `04_specular_razor_highlights.jpg`.
- **Saturated direct-lit color.** The preset is not red, not purple, not any color in albedo. All visible color comes from base-sky reflection + D-022 mood-tint on `scene.lightColor.rgb` + aurora-band reflections in the sky function. Direct albedo is `float3(0.02, 0.03, 0.05)`. The colored gels in `02_*` and `04_*` are studio photography, not palette directives.
- **Beat-driven primary motion.** Per `CLAUDE.md §Audio Data Hierarchy`, beat onsets are accents only; continuous energy (`*_energy_dev`) is the primary driver. The preset must route audio to visual layers via energy envelopes, not onset edges. Current routing post-round-65: spike height (`bass_energy_dev` continuous), aurora intensity (`drums_energy_dev_smoothed` 150 ms EMA), swell amplitude (`arousal`, slow). `Scripts/check_drums_beat_intensity.sh` enforces no `drums_beat` use in intensity scope.

- **Two beat-reactive layers responding to the same audio at similar magnitudes** (rounds 60/61 lesson). The pre-round-65 routing had both swell amplitude AND spike heights coupled to drums-related primitives at quarter-note / per-beat cadence. Visually this read as "competing rhythms" because two motions encoded the same beat info at different visual layers. Round 65 fix: route different audio timescales to different layers (slow `arousal` → swell; per-beat `bass_dev` → spike heights; orbit-paced `accumulated_audio_time` → aurora drift). Each visual layer gets one audio-timescale primitive, no double-coupling.
- **Decoration layers without articulated musical role.** A specific risk recurrence of Failed Approach #62 — Session 4 added Cassie-Baxter droplets, meso domain warp, and matID == 2 micro-normal perturbation, all of which had nominal audio routing but no listener-pairable musical moment and were reverted in Session 4.5 Phase 0. Before re-introducing any decoration layer (or adding new ones), the one-sentence check is: *"can a listener point at a moment in the music and identify the visual response THIS layer produces?"* If the layer cannot pass that test, do not add it.

## Constant-field premise (V.9 Session 4.5c Phase 1 round 50, 2026-05-16)

The preset's framing is "what if the ocean were made of ferrofluid?" That premise implies an ambient, permanent magnetic field — same way real oceans have ambient gravity and salinity. Under this premise:

- The Rosensweig spike **base lattice** (particle positions, wavelengths, propagation speeds, base shape) is a constant geometric property of the ocean's identity, regardless of whether music is playing or not.
- Music modulates **secondary dynamics**:
  - Spike HEIGHT (`bass_dev` → 0–35 % taller spikes — round 65 reactivation after rounds 50/63 had spike-height constant).
  - Swell amplitude (`arousal` → slow drift; round 65 removed the drums-coupling that was making the substrate beat-pump).
  - Aurora curtain (vocals → hue, drums → intensity, arousal → drift).
- The pre-round-50 routing (Leitl-style "magnetic field strength = audio amplitude" → spikes erupt with bass and collapse at silence) was borrowed from Leitl's tabletop demo and contradicts the ocean-scale premise. It produced a visualizer that read as a wavy sheet during quieter musical sections — wrong for "ferrofluid ocean."

**Geometric parameters as of round 65:**

The base lattice is the implementation of the "ferrofluid identity." Music response lives in the responsive layers: spike heights pulse with bass, swell amplitude drifts with arousal, aurora curtain rotates with arousal-paced orbit and changes hue with vocals/valence.

**History of attempted spike-height audio coupling experiments (rounds 56/60/61/63/65):** Several iterations on whether and how to make spike heights audio-reactive while preserving the constant-field premise. Round 56 added a subtle 10 % bass-dev term — too subtle to read. Round 60 bumped to 35 % — visible but competed with swell when swell was also drums-coupled. Round 61 gated to downbeat via `bar_phase01` — read as a defect because bar boundaries don't align with bass kicks musically. Round 63 reverted to constant. Round 65 reactivated at 35 % continuous AFTER removing drums-coupling from swell — with only one beat-reactive layer active, the spike pulse reads as intended response without competing motion. The lesson: don't route the same audio timescale to two different visual layers; assign each timescale to one layer.

## Audio routing notes

Specific audio→visual mappings as of round 65 (2026-05-18). Cite D-026 deviation primitives and D-019 stem warmup.

- **Spike height ← `stems.bass_energy_dev`** (round 65 — `1.0 + 0.35 × clamp(bass_dev, 0, 1)` in `fo_spike_strength`). Continuous response; spike heights pulse 0–35 % above baseline with bass intensity. The base lattice (radius, particle positions, propagation rates) remains constant; only the height multiplier modulates with bass. See "Constant-field premise" for the full history of this routing's experiments (rounds 50/56/60/61/63/65).
- **Swell amplitude ← `features.arousal` only** (round 65 — `0.4 + 0.6 × smoothstep(-0.5, 0.5, arousal)` in `fo_swell_scale`). Slow energy-driven drift. The previous `0.3 × drums_dev` per-beat accent term was removed in round 65 because the substrate beat-pumping competed with the bass-driven spike pulse — Matt's 2026-05-18T13-37-57Z review identified this as the visible "substrate tied to bass" effect. With only spike heights carrying the beat now, the swell becomes purely atmospheric body-of-liquid motion.
- **Gerstner wave propagation ← `features.time`** (round 58). Wall-clock seconds, monotonic. Waves cycle at 7–12 s natural periods (constant, regardless of music). Audio modulates wave AMPLITUDE via arousal (above), not wave PHASE.
- **Aurora curtain drift ← `features.accumulated_audio_time × features.arousal`** (D-127, round 61 base period). Implemented inline in `rm_ferrofluidSky`. `kCurtainBaseRevolutionSeconds = 2.5`; arousal mapped to [0.5, 1.0] speed factor; wall-clock revolutions ~25–37 s depending on arousal. `accumulated_audio_time` pauses at silence per MIRPipeline contract, so the curtain orbit pauses at silence too.
- **Aurora curtain hue ← `stems.vocals_pitch_hz`** normalized inline (perceptual log-scale over 80 Hz – 1 kHz, ±0.20 phase shift) **with `features.valence` mood fallback** below confidence 0.6 (D-127: drives palette phase) — implemented inline in `rm_ferrofluidSky`. Smooth crossfade between pitch and valence phases at confidence ∈ [0.5, 0.7] avoids hue popping at the confidence edge. Orbit-driven phase shift `0.35 × sin(curtainAzimuth × 0.5)` (round 53 — bumped from 0.10 for visible primary rotation through pink/green/purple).
- **Aurora curtain intensity ← `stems.drums_energy_dev_smoothed`** (150 ms τ EMA on the CPU side in `RenderPipeline.drawWithRayMarch`). Intensity = `2.5 + 2.0 × drumsSmoothed_clamped_1.5` (round 51 / round 53 tuning). Envelope-tracking, never edge-triggered — does not strobe on drum onsets.
- **Live-stems silence gate ← `totalStemEnergy = vocals + drums + bass + other` energies** (D-127: `smoothstep(0.02, 0.10, …)` fades the aurora out at silence) — implemented inline in `rm_ferrofluidSky`. On real-music captures the gate saturates to 1.0 immediately (Love Rehab `totalStemEnergy`: min 0.14 / mean 1.09); the gate is the safety net for prep states, intro-to-song boundaries, and source-app paused.
- **Aurora curtain coverage** (round 51 — wide-aurora geometry). `kCurtainElevation = 0`, `kCurtainStripeThickness = 0.70`, `kCurtainAzimuthFloor = -0.70`, `kCurtainAzimuthPeak = 0.80`. Covers most of the sky's upper hemisphere with bright neon palette, smoothly falling off at the edges. Substrate reflection picks up aurora content at many surface normals, not just narrow horizon-aligned ones.
- **Base sky values** (round 53 — Refn aesthetic crush). `lowSky = (0.002, 0.001, 0.004)`, `midSky = (0.020, 0.010, 0.030)`, `highSky = (0.010, 0.006, 0.018)`. Crushed to near-black; the saturated aurora is the only significant chromatic content. The bright/dark sky contrast produces the dramatic chiaroscuro on mirror cone surfaces.
- **Base sky tint ← D-022 mood valence** (matID == 2 path) — implemented (`rm_ferrofluidBaseSky` multiplies its three-stop gradient by `scene.lightColor.rgb`). Negative valence → deeper purple / cooler; positive valence → warmer purple drifting toward magenta. The shift propagates through every mirror-reflected pixel of the substrate. Note: matID == 2 bypasses `rm_finishLightingPass` under D-126 and the sky function does the equivalent work.
- **Thin-film thickness ← arousal** (matID == 3 fallback path only). Not active in matID == 2 production path. 220 nm baseline ± 40 nm range → effective [180, 260] nm if/when the fallback path is exercised.

## Render pipeline history

The preset's `passes` array (`["ray_march", "post_process"]`) is stable, but which G-buffer fragment runs has changed:

- **Sessions 1–3 (2026-05-13 → 2026-05-15):** SDF G-buffer fragment (`raymarch_gbuffer_fragment` in the preamble) calling preset-defined `sceneSDF`. Standard ray-march per-pixel SDF evaluation.
- **Session 4.5c Phase 1 Step B (round ~35, 2026-05-15):** added MESH G-buffer path. `VisualizerEngine+Presets.swift` constructed a `FerrofluidMesh` (tessellated 512×512 quad mesh) and registered it via `pipeline.setMeshGBufferEncoder`. When set, `RayMarchPipeline.render` took the `runMeshGBufferPass` branch instead of `runGBufferPass`. The mesh path computed normals via vertex displacement and per-pixel heightmap sampling.
- **Round 57 (2026-05-17):** MESH path retired. Diagnostic discovery: test fixtures took the SDF branch (no mesh encoder set) while live took the MESH branch — a structural test/prod gap that hid the mesh path's normal-computation defects (artifacts read as "scoops" on cone silhouettes). Round 56's SDF Lipschitz fix worked beautifully in fixtures but didn't reach live because live wasn't using the SDF path. Round 57 removed the mesh-encoder wire-up in `VisualizerEngine+Presets.swift` — live now uses the SDF path. The `FerrofluidMesh.metal` shader and `FerrofluidMesh.swift` Swift code remain in tree for a future increment that may address the mesh path's normal computation properly.
- **Round 57 (same):** fixture helper `renderDeferredRayMarch` gained a `useMeshPath: Bool` parameter (default `false`, matching new live behavior) that constructs a `FerrofluidMesh` and binds the encoder when set to `true`. This closes the structural test/prod branch gap permanently — future fixtures can verify both paths.
